# VOĆNJAK APP — VISION & STRATEGY

## PURPOSE

Build a simple, fast, orchard-first app that becomes a real, usable product.

---

## TARGET

- daily-use tool
- iPhone-first UX
- App Store ready
- monetizable (freemium)

---

## POSITIONING

Not:
- generic gardening app
- complex farm software

But:
- simple orchard app focused on real work

---

## CORE VALUE

User must be able to:

1. see what to do today
2. quickly log activity
3. understand plant state
4. track work over time

---

## CORE SYSTEM PRINCIPLES

The product is based on a deterministic model:

- activities are the only source of truth
- plans are predefined domain knowledge (not user-managed)
- plan state is always derived from activities
- no hidden logic, no assumptions, no heuristics

The system must always be:
- predictable
- explainable
- consistent

---

## UX PHILOSOPHY

- minimal taps
- no thinking required
- one-handed usage
- fast interaction over visual complexity

---

## DEVELOPMENT STRATEGY

1. core workflow first
2. stable data model
3. deterministic logic
4. UX polish
5. then expansion

---

## PRODUCT EVOLUTION

The app evolves from:

→ activity logger

to:

→ orchard assistant that tells user what to do

But:
- intelligence must be rule-based first
- no AI or complexity before strong core

---

## BIGGEST RISK

Turning this into:
- a complex system
- a generic tool
- an over-engineered product

Instead of:
- a fast, useful, daily tool

---

## FINAL RULE

Keep it:
- simple
- practical
- focused on real user value
